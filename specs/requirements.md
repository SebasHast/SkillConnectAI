# Requerimientos

## Funcionales

### RF01
El usuario puede escribir su nombre y diagnóstico.

### RF02
El sistema debe recomendar una especialidad médica automáticamente.

### RF03
El usuario puede seleccionar:
- médico
- fecha
- hora

### RF04
El sistema debe validar disponibilidad.

### RF05
El sistema no debe permitir conflictos de horario.

### RF06
El usuario puede registrar citas médicas.

### RF07
El sistema debe listar citas registradas.

### RF08
El usuario puede eliminar citas.

### RF09
El usuario puede filtrar citas por especialidad.

### RF10
El sistema debe mostrar estadísticas dinámicas.

---

## No Funcionales

### RNF01
La interfaz debe ser responsive.

### RNF02
El sistema debe responder en menos de 2 segundos.

### RNF03
El diseño debe ser intuitivo y accesible.

### RNF04
El código debe ser mantenible.

### RNF05
La disponibilidad debe actualizarse dinámicamente.

### RNF06
La aplicación debe funcionar sin frameworks frontend.
